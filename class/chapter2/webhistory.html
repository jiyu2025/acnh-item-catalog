개요

웹은 1989년 팀 버너스-리(Tim Berners‑Lee)가 제안한 월드 와이드 웹 구상에서 출발했습니다. 웹은 문서를 서로 연결하는 하이퍼텍스트 개념을 기반으로 하며, 브라우저와 서버, HTML 같은 표준을 통해 빠르게 확산되었습니다.

연표

  1. 1989–1991: 발명과 초기 구현

    팀 버너스‑리의 웹 제안과 최초의 웹 서버 및 브라우저(정보 저장소: info.cern.ch). HTML의 초기 버전이 사용되기 시작.

  2. 1993: 그래픽 브라우저의 등장

    Mosaic 브라우저의 등장으로 일반 사용자가 웹을 쉽게 접할 수 있게 되어 급격한 보급이 시작됨.

  3. 1994: W3C 설립

    웹 표준화를 위해 월드 와이드 웹 컨소시엄(W3C)이 설립되어 HTML, CSS, 접근성 등의 표준 작업이 조직적으로 진행됨.

  4. 1995–2000: 브라우저 경쟁과 표준의 필요성

    Netscape, Internet Explorer 등의 경쟁과 함께 다양한 확장들이 생겨났고 표준 준수의 중요성이 커짐.

  5. 1996: CSS 제안

    스타일시트(CSS) 개념이 도입되어 문서의 표현과 구조를 분리하는 기반이 마련됨.

  6. 2004–2006: 웹 2.0과 동적 웹

    사용자 참여형 서비스(블로그, 소셜 미디어)가 확산되고 AJAX 같은 기술로 대화형 웹 애플리케이션이 등장.

  7. 2008–현재: 모바일 우선과 HTML5

    스마트폰 보급으로 모바일 중심의 웹 설계가 중요해졌고, HTML5와 관련 표준들이 멀티미디어와 풍부한 기능을 네이티브하게 지원하게 됨.

주요 이정표

  • HTML: 문서 구조를 정의하는 마크업 언어.
  • HTTP: 웹의 통신 규약.
  • 브라우저: 사용자와 웹을 연결하는 소프트웨어.
  • CSS: 스타일과 레이아웃을 분리.
  • 웹 표준과 접근성: 모든 사용자에게 동등한 접근을 보장.
  • 프로그레시브 웹 앱(PWA): 네이티브 앱과 유사한 경험 제공.

현대의 웹

오늘날의 웹은 성능, 접근성, 보안, 개인정보 보호, 모바일 호환성 등을 중요한 가치로 삼습니다. 또한 오픈 웹 표준을 기반으로 다양한 기기와 플랫폼에서 일관된 경험을 제공하려는 노력이 계속되고 있습니다.

학습 팁

  • 초기 웹 역사와 info.cern.ch를 살펴보면 원형을 이해하는 데 도움이 됩니다.
  • W3C 문서와 WHATWG 사양을 참고하면 최신 표준을 파악할 수 있습니다.
